442 # The overall hierarchy is
443 # `checkbounds(A, I...)` ->
444 # `checkbounds(Bool, A, I...)` ->
445 # `checkbounds_indices(Bool, IA, I)`, which recursively calls
446 # `checkindex` for each dimension
447 #
448 # See the "boundscheck" devdocs for more information.
449 #
450 # Note this hierarchy has been designed to reduce the likelihood of
451 # method ambiguities. We try to make `checkbounds` the place to
452 # specialize on array type, and try to avoid specializations on index
453 # types; conversely, `checkindex` is intended to be specialized only
454 # on index type (especially, its last argument).
